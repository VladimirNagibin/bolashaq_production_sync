# Работа с товарами: добавление свойств, брендов, источников

## Добавление свойства в товар

1. **Добавить свойство товара в Битрикс24**
   - **Простое свойство** – хранит пару `value_id`, `value`.
   - **Составное свойство** – хранит `value_id`, `value`: {`TEXT`, `TYPE`}.

2. **На сервере** в `schemas.product_schemas.BaseProduct` добавить поле с указанием `validation_alias = AliasChoices(...)`, где перечисляются имена поля в Битрикс.

3. **В `schemas.fields.FIELD_PRODUCT`** указать имя поля в Битрикс:
   - для простого свойства – в словаре `dict_none_str`;
   - для составного – в словаре `dict_none_dict`.

4. **В `schemas.fields.FIELD_PRODUCT_ALT`** указать имя поля из схемы на сервере:
   - для простого свойства – в `dict_none_str`;
   - для составного – в `dict_none_dict`.

## Добавление бренда

1. Добавить новое значение в **список значений поля «Бренд»** в Битрикс24.
   Запомнить его **числовое представление** (ID).

2. На сервере в файле `schemas.enums.BrandEnum` добавить:
   - значение (числовой ID из Битрикс);
   - код (строковый идентификатор);
   - оригинальное название для отображения в `def display_names`.

**Пример:**
```python
class BrandEnum(IntEnum):
    NEW_BRAND = 123  # числовой ID из Битрикс
    # ...
```

## Добавление источника

1. Добавить новое значение в **список значений поля «Источники»** в Битрикс24.
   Запомнить его **числовое представление** (ID).

2. На сервере в файле `schemas.enums.SourcesProductEnum` добавить:
   - код (строковый идентификатор);
   - значение - оригинальное название для отображения;
   - числовой ID из Битрикс в `def mapping_bitrix_id`.

**Пример:**
```python
class SourcesProductEnum(StrEnum):
    NEW_SOURCE = "new source"  # название для отображения
    # ...
```
